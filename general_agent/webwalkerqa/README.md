# WebWalkerQA T³ Proof of Concept

Answers the core research question:
**Given a fixed compute budget, does parallel diverse threading (T³) outperform sequential token scaling (s1)?**

## What's Here

```
webwalkerqa/
├── configs.py          # Experiment matrix: A1–C3, Oracle
├── dataset.py          # WebWalkerQA loader (HuggingFace or local JSON)
├── eval.py             # Exact Match with normalization
├── search.py           # Serper web search wrapper
├── llm.py              # LiteLLM async wrapper
├── methods/
│   ├── base.py         # BaseMethod, MethodResult, TurnLog
│   ├── s1.py           # Sequential scaling (single thread)
│   └── t3_fixed.py     # T³ Fixed (k parallel threads + diversity seeds)
├── runner.py           # Per-question runner with resume + oracle
├── experiment.py       # Main CLI entry point
├── analyze.py          # Stats table + Figure 1
└── scripts/
    ├── run_poc.sh          # Run a single config
    └── run_full_matrix.sh  # Run the entire experiment matrix
```

## Quick Start

```bash
cd general_agent

# Set up env
cp .env-example .env  # fill in SERPER_API_KEY, OPENAI_API_KEY, etc.

# Run a quick test (10 questions, verbose)
MAX_EXAMPLES=10 VERBOSE=1 ./webwalkerqa/scripts/run_poc.sh A2 openai/gpt-4o-mini

# Run a specific config
./webwalkerqa/scripts/run_poc.sh B2 openai/gpt-4o-mini

# Run all s1 configs
./webwalkerqa/scripts/run_poc.sh s1 openai/gpt-4o-mini

# Run all T³ configs
./webwalkerqa/scripts/run_poc.sh t3 openai/gpt-4o-mini

# Run full matrix (generates Figure 1)
MODEL=openai/gpt-4o-mini ./webwalkerqa/scripts/run_full_matrix.sh

# Analyze results
python3 -m webwalkerqa.analyze --results-dir results/webwalkerqa
```

## Extending to T³ Dynamic / T³ DPP

The method hierarchy is designed for extension. To add T³ Dynamic:

1. Create `methods/t3_dynamic.py` subclassing `T3FixedMethod`
2. Override `_generate_seeds()` to call the LLM instead of using the fixed pool
3. Register it in `methods/__init__.py`

To add T³ DPP:
1. Create `methods/t3_dpp.py` subclassing `T3FixedMethod`
2. Override `_generate_seeds()` with DPP candidate selection
